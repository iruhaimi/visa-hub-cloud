

# خطة تعديل صلاحيات المشرف

## الهدف
تقييد قائمة المشرف (admin غير المالك) لتظهر فقط 6 عناصر، وإخفاء تبويب "الصلاحيات" من صفحة الإعدادات عنه.

## القائمة المطلوبة للمشرف
| العنصر | الحالة |
|--------|--------|
| لوحة التحكم | ✅ يظهر |
| الطلبات | ✅ يظهر |
| العروض الخاصة | ✅ يظهر |
| طلبات الاسترداد | ✅ يظهر |
| طلبات نقل الطلبات | ✅ يظهر |
| الإعدادات (بدون الصلاحيات) | ✅ يظهر |
| المستخدمين | ❌ مالك فقط |
| إدارة الهيرو | ❌ مالك فقط |
| إدارة الفوتر | ❌ مالك فقط |
| طلبات فتح الحساب | ❌ مالك فقط |
| سجل محاولات الدخول | ❌ مالك فقط |
| سجل الوصول للمستندات | ❌ مالك فقط |

## التغييرات

### 1. تعديل `src/components/admin/AdminLayout.tsx`
تغيير شرط `show` للعناصر التالية لتصبح `isSuperAdmin` بدلاً من `hasPermission(...)`:
- **المستخدمين** → `isSuperAdmin`
- **إدارة الهيرو** → `isSuperAdmin`
- **إدارة الفوتر** → `isSuperAdmin`
- **طلبات فتح الحساب** → `isSuperAdmin`
- **سجل محاولات الدخول** → `isSuperAdmin`
- **سجل الوصول للمستندات** → `isSuperAdmin`

تبسيط شروط العناصر المتاحة للمشرف:
- **الطلبات** → `isAdmin` (بدون شرط صلاحية إضافي)
- **العروض الخاصة** → `isAdmin`
- **طلبات الاسترداد** → `isAdmin`
- **طلبات نقل الطلبات** → `isAdmin`
- **الإعدادات** → `isAdmin`

### 2. تعديل `src/pages/admin/Settings.tsx`
- تمرير `usePermissions()` لمعرفة إن كان المستخدم مالك
- إخفاء تبويب "الصلاحيات" من `TabsList` إذا لم يكن `isSuperAdmin`
- إخفاء `TabsContent` الخاص بالصلاحيات كذلك

### 3. حماية المسارات (Routes)
لا حاجة لتعديل المسارات في `App.tsx` لأن `ProtectedRoute` يتحقق من دور `admin` فقط، والتحكم يتم عبر القائمة الجانبية. لكن يُفضل إضافة حماية إضافية داخل الصفحات المقيدة (المستخدمين، الهيرو، إلخ) بحيث إذا حاول مشرف الوصول مباشرة عبر الرابط يتم توجيهه للوحة التحكم. سيتم ذلك بإضافة تحقق `isSuperAdmin` في بداية كل صفحة مقيدة مع إعادة توجيه.
