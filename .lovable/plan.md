

# خطة: ترقية المستخدم وتأكيد صفحة إدارة الصلاحيات

## الوضع الحالي

**أخبار سارة!** صفحة إدارة المستخدمين **موجودة ومكتملة بالفعل** في لوحة التحكم:
- المسار: `/admin/users`
- تظهر في القائمة الجانبية تحت "المستخدمين"
- تتيح البحث عن المستخدمين وإضافة/حذف الصلاحيات

## المهمة المطلوبة

| المهمة | الحالة |
|--------|--------|
| صفحة إدارة الصلاحيات | موجودة ✓ |
| ترقية i.alruhaimi@gmail.com إلى admin | **مطلوب التنفيذ** |

## الإجراء المطلوب

### ترقية المستخدم `i.alruhaimi@gmail.com`

سأقوم بإضافة صلاحية `admin` للمستخدم عبر إدراج سجل جديد في جدول `user_roles`:

```text
المستخدم: i.alruhaimi@gmail.com
user_id: a3869d1a-c351-4c62-922d-c9b5cec30fee
الصلاحية الجديدة: admin
```

## التفاصيل التقنية

### استعلام قاعدة البيانات
```sql
INSERT INTO user_roles (user_id, role)
VALUES ('a3869d1a-c351-4c62-922d-c9b5cec30fee', 'admin');
```

### النتيجة المتوقعة
بعد التنفيذ، المستخدم `i.alruhaimi@gmail.com` سيمتلك صلاحيتين:
1. `customer` (موجودة)
2. `admin` (جديدة)

وسيتمكن من:
- الوصول لبوابة الموظفين `/staff`
- الدخول للوحة تحكم المشرف `/admin`
- إدارة صلاحيات المستخدمين الآخرين من `/admin/users`

## ميزات صفحة إدارة المستخدمين الموجودة

| الميزة | الوصف |
|--------|-------|
| عرض المستخدمين | جدول بجميع المستخدمين مع بياناتهم |
| البحث | بالاسم أو رقم الجوال أو الجنسية |
| عرض الصلاحيات | شارات ملونة (عميل/وكيل/مشرف) |
| إضافة صلاحية | زر لإضافة صلاحية جديدة لأي مستخدم |
| حذف صلاحية | النقر على الشارة لحذفها |
| تحديث | زر لإعادة تحميل القائمة |
